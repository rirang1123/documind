# DocuMind v0.4.4 Release Report

**날짜**: 2026-03-11
**버전**: 0.4.4
**주요 변경**: 파일 자동 분류 강화 + 디스크 저장소 동기화 수정

---

## 변경 사항

### 1. 파일 자동 분류 대폭 강화

#### 키워드 패턴 확장 (`classifyService.ts`)
- **보고 카테고리**: 기존 10개 → 30+개 키워드
  - 추가: 실적, 성과보고, 월간, 주간, 일일, 데일리, 연간, 분기, 결산, 동향, 추이, KPI, 대시보드, 업무보고, 현장보고, 출장보고, 점검보고 등
- **회의 카테고리**: 기존 9개 → 20+개 키워드
  - 추가: 워크숍, 세미나, 브리핑, 발표자료, 프레젠테이션, 토론, 토의, 협의, 간담회, 조회 등
- **기획 카테고리**: 기존 10개 → 25+개 키워드
  - 추가: 제안서, proposal, 전략수립, 방안, 추진계획, 액션플랜, 시행계획, 연구계획, 개발계획, 마케팅계획 등
- **평가 카테고리**: 기존 9개 → 20+개 키워드
  - 추가: 성과평가, 인사평가, 업무평가, 품질검사, audit, 감사, 진단, 테스트결과, 검증, 승인 등
- **참고자료 카테고리**: 기존 8개 → 20+개 키워드
  - 추가: tutorial, 안내서, 핸드북, 사양서, 규격, 표준, 절차서, 교육자료, 기술문서, API문서 등
- **계약 카테고리**: 기존 9개 → 20+개 키워드
  - 추가: 합의서, 발주서, 납품, 입찰, 견적서, 비밀유지, NDA, 라이선스 등
- **재무 카테고리**: 기존 12개 → 25+개 키워드
  - 추가: 청구서, 영수증, 결제, 정산, 급여, 세무, 감가상각, 자산, 현금흐름, 원가, 거래명세, 전표 등

#### AI Fallback 분류 추가
- **`smartClassify()` 함수 신규**: 규칙 기반 분류가 "기타"일 때 AI 자동 분류 시도
- AI 제공자 설정 시 `aiService.classifyDocument()` 호출하여 지능적 분류
- AI 분류 결과를 `FolderCategory`로 자동 매핑
- AI 미설정 또는 실패 시 graceful fallback ("기타" 유지)
- AI 분류 시 태그도 자동 생성하여 파일에 저장

### 2. 파일 디스크 저장소 동기화 수정

#### 웹 파일 업로드 (`classifyAndAddWebFile`)
- **이전**: `path: f.name` (파일명만 저장, 디스크에 복사 안 됨)
- **수정**: 프로젝트 디스크 폴더의 분류 카테고리 하위에 실제 파일 복사
  - 텍스트: `writeTextFile()`, 바이너리: `writeFile()` 사용
  - `path` 필드에 실제 디스크 저장 경로 설정
  - 폴더 미존재 시 자동 생성

#### Tauri 드래그앤드롭 업로드 (`classifyAndAddTauriFile`)
- **이전**: `path: filePath` (원본 경로만 참조, 프로젝트 폴더에 복사 안 됨)
- **수정**: 원본 파일을 프로젝트 디스크 폴더의 분류 카테고리 하위로 복사
  - 원본과 대상이 같은 경우 복사 스킵
  - `path` 필드에 복사된 위치의 실제 경로 설정

#### 에디터 문서 저장 (`SaveToProjectDialog`)
- **이전**: `path: docName.trim()` (문서명만, 디스크 저장은 별도)
- **수정**: 디스크 저장 후 실제 경로를 `path` 필드에 설정
- `smartClassify()` 사용으로 분류 정확도 향상
- AI 태그 자동 적용

### 3. 저장 흐름 정리

```
파일 업로드/저장 → smartClassify(제목, 내용)
  ├─ 규칙 매칭 성공 → 해당 카테고리
  └─ 규칙 매칭 실패 ("기타")
     ├─ AI 제공자 설정됨 → AI 분류 + 태그 생성
     └─ AI 미설정 → "기타" 유지

→ ensureCategoryFolder() → 분류 폴더 생성 (DB + 디스크)
→ 파일 복사 (디스크 분류 폴더에)
→ DocumentFile 저장 (DB, path=실제 디스크 경로)
```
