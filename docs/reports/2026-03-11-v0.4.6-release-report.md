# DocuMind v0.4.6 Release Report

**날짜**: 2026-03-11
**버전**: 0.4.6
**주요 변경**: 프로젝트 삭제 시 디스크 데이터 제거 + 데이터 전체 초기화 기능

---

## 변경 사항

### 1. 프로젝트 삭제 시 디스크 폴더 제거 (`useAppStore.ts`)

#### 문제
- `deleteProject`가 IndexedDB 레코드만 삭제하고, 디스크의 프로젝트 폴더(`Documents/DocuMind/{프로젝트명}/`)는 남아있었음
- 소프트웨어 재설치 시 기존 폴더가 남아 새 프로젝트 생성에 충돌

#### 해결
- `deleteProject(id, deleteFromDisk?)` — 두 번째 매개변수로 디스크 삭제 여부 선택
- `deleteFromDisk=true`일 때: `@tauri-apps/plugin-fs`의 `remove(path, { recursive: true })`로 프로젝트 폴더 전체 삭제
- DB 레코드(projects, files, folders) 삭제는 기존과 동일

### 2. Sidebar 프로젝트 삭제 UI (`Sidebar.tsx`)

#### 추가된 기능
- 프로젝트 **우클릭** → 컨텍스트 메뉴 → "프로젝트 삭제"
- 삭제 확인 다이얼로그:
  - **전부 삭제 (DB + 저장소 폴더)**: IndexedDB + 디스크 폴더 모두 제거
  - **DB만 삭제 (저장소 폴더 유지)**: IndexedDB만 제거, 디스크 파일은 보존
  - **취소**: 아무것도 하지 않음

### 3. 데이터 전체 초기화 (`SettingsView.tsx`)

#### 문제
- 소프트웨어 삭제 후 재설치 시 IndexedDB 데이터와 디스크 폴더가 그대로 남아있음
- WebView2의 IndexedDB는 Windows 앱 삭제와 무관하게 유지됨

#### 해결: "데이터 전체 초기화" 섹션 추가
- 설정 > 일반 탭에 빨간색 "데이터 전체 초기화" 섹션 추가
- 안전장치: "초기화" 텍스트를 입력해야만 버튼 활성화
- 실행 시:
  1. 저장소 디스크 폴더 전체 삭제 (`Documents/DocuMind/` 또는 사용자 지정 경로)
  2. IndexedDB 전체 삭제 (`db.delete()`)
  3. 앱 자동 재시작 (`relaunch()`)

#### 사용 시나리오
- 소프트웨어 삭제 전: 이 기능으로 모든 데이터 정리 → 삭제 → 깨끗한 재설치
- 데이터 꼬임 해결: 초기화 후 백업 파일로 복원
