# DocuMind v0.6.1 릴리스 리포트

**작성일**: 2026-03-13
**버전**: 0.6.0 → 0.6.1

---

## 1. 작업 요약

### 1-1. CI/CD .sig 파일 미생성 문제 해결

**문제**: v0.6.0 CI/CD 릴리스 시 빌드(`.exe`, `.msi`)는 성공하지만 `.sig` 서명 파일이 전혀 생성되지 않음

**디버깅 과정**:
1. 워크플로우에 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` 빈 문자열 기본값 추가 → 실패
2. 디버그 스텝(번들 파일 리스팅) 추가하여 `.sig` 파일 존재 여부 확인 → `.sig` 0개 확인
3. 서명 키 재생성 + GitHub Secrets 갱신 → 실패
4. **근본 원인 발견**: `tauri.conf.json`의 `bundle` 섹션에 `createUpdaterArtifacts` 설정 누락

**해결**: `bundle.createUpdaterArtifacts: "v1Compatible"` 추가
- Tauri v2에서 이 설정 없이는 `TAURI_SIGNING_PRIVATE_KEY`가 있어도 서명 파일을 생성하지 않음

**수정 파일**:
| 파일 | 변경 내용 |
|------|----------|
| `.github/workflows/release.yml` | 패스워드 기본값 `\|\| ''`, 디버그 스텝, 전체 파일 리스팅 |
| `.github/workflows/release-test.yml` | 동일 |
| `src-tauri/tauri.conf.json` | `createUpdaterArtifacts: "v1Compatible"` 추가, pubkey 갱신 |
| `src-tauri/tauri.test.conf.json` | 동일 |

### 1-2. 서명 키 재생성

- `npx tauri signer generate` 로 새 키 쌍 생성 (빈 비밀번호)
- GitHub Secrets 갱신: `TAURI_SIGNING_PRIVATE_KEY`, `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`
- `tauri.conf.json`, `tauri.test.conf.json`의 `pubkey` 갱신

### 1-3. fs:scope 전체 경로 허용

**문제**: `D:\test docs` 등 외부 드라이브에서 프로젝트 생성 시 `forbidden path` 오류

**원인**: `src-tauri/capabilities/default.json`의 `fs:scope`에 C드라이브 사용자 폴더만 허용

**해결**: `{ "path": "**" }` 추가하여 모든 경로 접근 허용

### 1-4. 인앱 자동 업데이트 활성화

**문제**: `UpdateChecker` 컴포넌트가 구현되어 있었으나 `App.tsx`에 마운트되지 않음

**해결**: `App.tsx`에 `<UpdateChecker />` 추가
- 앱 시작 시 자동으로 최신 버전 확인
- 새 버전 감지 시 우하단 토스트 알림 (다운로드 + 설치 + 재시작)
- 설정에서도 수동 확인 가능 (`UpdateSection`)

### 1-5. README 변경 이력 추가

- README.md 하단에 v0.1.4 ~ v0.6.1 전체 변경 이력 추가
- 사용자 기능 중심 정리 (CI/CD, 인프라 내용 제외)
- 자동 업데이트 섹션 제거 (미검증)

### 1-6. 개인정보처리방침 페이지

- `docs/privacy-policy.html` 생성
- GitHub Pages 배포: https://rirang1123.github.io/documind/privacy-policy.html
- Google OAuth 인증 요청 시 필요 (현재는 유저별 개인 등록 방식이므로 불필요)

## 4. 참고 사항

- **Tauri v2 서명 필수 설정**: `bundle.createUpdaterArtifacts: "v1Compatible"` — 이 설정 없이는 환경변수가 있어도 `.sig` 파일이 생성되지 않음
- **로컬 빌드 시 서명**: `TAURI_SIGNING_PRIVATE_KEY` 환경변수 필요 (`~/.tauri/documind.key`)
- **Google OAuth 인증**: 유저별 개인 등록 방식이므로 Google 앱 인증 불필요. 각 유저가 자기 OAuth 클라이언트의 테스트 사용자로 등록하면 됨
