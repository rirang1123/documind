## 1. 앱 전용 폴더 내보내기 시스템 (신규)

### 배경
기존에는 OS 네이티브 저장 다이얼로그(`save()`)를 사용했으나, 앱이 관리하는 전용 폴더(`문서/DocuMind/`) 내에서 커스텀 UI로 저장 위치를 선택하도록 변경.

### 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `src/services/document/exportService.ts` | OS `save()` 다이얼로그 제거. `getExportBaseDir()`, `ensureExportDir()`, `listFolderContents()`, `createFolder()` 추가. `exportDocument()`에 `targetDir` 파라미터 추가 |
| `src/components/editor/ExportLocationDialog.tsx` | **신규**. breadcrumb 네비게이션, 폴더/파일 목록, 새 폴더 생성, 현재 경로 표시, "여기에 저장" 버튼 포함 커스텀 UI |
| `src/components/editor/ExportDialog.tsx` | 2단계 흐름: 형식/이름 선택 → "저장 위치 선택" → ExportLocationDialog |
| `src/services/document/pptService.ts` | OS 다이얼로그 제거, `buildPptx()` / `generatePptFromDocument()`에 `targetDir` 파라미터 추가 |
| `src/components/editor/ppt/PptSlideEditor.tsx` | PPT 생성 시 ExportLocationDialog 연동 |
| `src/components/editor/PptGenerateDialog.tsx` | AI PPT 생성 시 ExportLocationDialog 연동 |

## 2. AI 검수 기능 버그 수정

### CTO 분석 결과 — 근본 원인 4건

| # | 원인 | 영향 |
|---|------|------|
| 1 | **Google Gemini API에서 system 메시지 완전 누락** — `chatGoogle()`이 `system` role을 필터링하여 검수 지침이 AI에 전달 안 됨 | Google 사용자: 검수 결과 항상 0건 |
| 2 | **JSON 파싱 실패 시 `catch {}` 무음 처리** — 에러를 삼키고 빈 배열 반환 | 파싱 오류 시 디버깅 불가 |
| 3 | **검토 완료 후 UI 피드백 없음** — 결과 0건이면 초기 상태로 돌아가 "실행만 됐다가 꺼지는" 것처럼 보임 | UX 혼란 |
| 4 | **프롬프트가 약함** — 구체적 한국어 교정 예시 없음, 분석 범위 4000자로 제한 | 실질적 교정 부족 |

### 수정 내용

| 파일 | 수정 |
|------|------|
| `src/services/ai-provider/aiService.ts` | Google API에 `systemInstruction` 필드로 system 프롬프트 전달 |
| `src/services/document/reviewService.ts` | 프롬프트 전면 개선(구체적 맞춤법/띄어쓰기 예시, 분석범위 6000자), 마크다운 래핑 JSON 파싱 대응, `console.error` 로깅 추가, 문제없으면 빈 배열 반환(억지 제안 금지) |
| `src/components/editor/ReviewPanel.tsx` | "검토 완료 — 문제 없음" 상태 표시, "총 N건 발견 / M건 수정됨" 요약, "다시 검토하기" 버튼, 수정 적용 카운터 |

## 3. 임시 저장 문서 시스템 (신규)

### 배경
기존에는 localStorage 기반 단일 자동저장만 존재. 여러 문서를 임시 저장하고 다시 열어 수정/완료/내보내기할 수 있는 시스템 필요.

### 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `src/types/index.ts` | `Draft` 인터페이스 추가 (id, title, content, createdAt, updatedAt) |
| `src/services/db.ts` | Dexie v2 스키마 — `drafts` 테이블 추가 |
| `src/stores/useAppStore.ts` | `loadDrafts`, `saveDraft`, `updateDraft`, `deleteDraft`, `openDraft` + `editingDraftId` 상태, `activeView`에 `'drafts'` 추가 |
| `src/components/drafts/DraftsView.tsx` | **신규**. 임시저장 문서 목록/검색/열기/내보내기/삭제 UI |
| `src/components/editor/EditorView.tsx` | 에디터 헤더에 "임시 저장" 버튼 추가, 드래프트 수정 시 자동 업데이트 |
| `src/components/layout/Sidebar.tsx` | "임시 저장" 네비게이션 항목 추가 |
| `src/components/layout/MainLayout.tsx` | `DraftsView` 라우팅 추가 |
| `src/App.tsx` | 앱 시작 시 `loadDrafts()` 호출 |

## 4. Tauri 파일시스템 권한 수정 (치명적 버그 수정)

### 근본 원인
`capabilities/default.json`에 `fs:default`만 설정 → 앱 전용 디렉토리(AppData 등)만 허용, `documentDir()`(문서 폴더) 접근 전면 차단됨.

모든 폴더 생성, 내보내기, 파일 저장이 권한 오류로 실패.

### 수정

| 파일 | 수정 |
|------|------|
| `src-tauri/capabilities/default.json` | `$DOCUMENT` 경로에 mkdir, readDir, exists, writeFile, writeTextFile, readFile, stat 권한 명시적 추가 |
| `src/components/editor/ExportLocationDialog.tsx` | 하드코딩 `\\` 경로 구분자 → Tauri `join()` 사용, 현재 경로 표시 UI 추가, 에러 메시지 상세화 |
| `src/services/document/exportService.ts` | 모든 함수에 try/catch + 구체적 에러 메시지 + console.error 로깅 |
