## 1. 프로젝트 폴더 생성 수정 (버그 수정)

### 근본 원인
- `NewProjectDialog`에서 `path: ''`(빈 문자열)로 프로젝트 생성 → 디스크에 폴더 미생성
- `getExportBaseDir()`가 설정의 `storagePath`를 무시하고 항상 `documentDir()/DocuMind/` 고정

### 수정 내용

| 파일 | 변경 내용 |
|------|-----------|
| `src/services/document/exportService.ts` | `getExportBaseDir()` — settings.storagePath 우선 사용. `createProjectFolder()` 신규 — 프로젝트명으로 디스크 폴더 생성(특수문자 치환). `createSubFolderOnDisk()` 신규 — 하위 폴더 디스크 생성 |
| `src/components/project/NewProjectDialog.tsx` | `createProjectFolder()` 호출하여 실제 디스크 폴더 생성, 생성된 경로를 `project.path`에 저장, 에러 처리/로딩 상태 추가 |
| `src/components/editor/ExportLocationDialog.tsx` | 프로젝트 선택 시 해당 프로젝트 폴더를 기본 경로로 사용 |

### 폴더 구조

```
저장소 경로 (설정 또는 기본: 내 문서/DocuMind/)
  ├── 프로젝트A/          ← 프로젝트 생성 시 자동 생성
  │   ├── 기획/           ← 자동 분류 or 수동 생성
  │   ├── 보고/
  │   └── 문서.docx       ← 내보내기 시 저장
  └── 프로젝트B/
```

## 2. 문서 저장 기능 신설 (신규 기능)

### 배경
기존에는 "임시 저장"(Draft → IndexedDB)과 "내보내기"(파일 시스템)만 존재. 프로젝트 내 문서로 정식 저장하는 기능 없음.

### 변경 내용

| 파일 | 변경 내용 |
|------|-----------|
| `src/components/editor/SaveToProjectDialog.tsx` | **신규**. 프로젝트/폴더 선택 UI, 문서 이름 입력, 자동 분류 폴더 생성, 신규 폴더 생성(카테고리 지정), IndexedDB에 DocumentFile로 저장 |
| `src/components/editor/EditorView.tsx` | "저장" 버튼(파란색, 메인) + "다른 위치에 저장" 버튼 추가. 기존 "임시 저장" 버튼 제거. 빠른 저장(이미 저장된 파일 즉시 업데이트) 구현 |

### 저장 흐름

| 버튼 | 동작 |
|------|------|
| **저장** | 이미 저장된 파일이면 즉시 업데이트. 처음이면 SaveToProjectDialog 열림 |
| **다른 위치에 저장** | 항상 SaveToProjectDialog 열어 프로젝트/폴더 선택 |
| **내보내기** | 기존과 동일 — docx/pdf/txt/html로 파일 시스템에 출력 |

### 자동 분류 로직
문서 **제목 + 본문 내용**을 함께 분석하여 카테고리 자동 감지. 제목에 3배 가중치 부여. 본문은 앞 2000자까지 분석. 각 카테고리별 키워드 매칭 점수가 가장 높은 카테고리로 분류.

| 카테고리 | 감지 키워드 |
|----------|------------|
| 기획 | 기획, 계획, 전략, plan, 목표, 로드맵, 일정, schedule, milestone |
| 평가 | 평가, 심사, 리뷰, review, 검토, 피드백, 성과 |
| 보고 | 보고, 리포트, report, 결과, 분석, 요약, 현황, 통계 |
| 참고자료 | 참고, 자료, reference, 가이드, 매뉴얼, 문헌 |
| 회의 | 회의, 미팅, meeting, agenda, 회의록, 안건, 참석자 |
| 계약 | 계약, contract, 협약, MOU, 약정, 조항, 체결 |
| 재무 | 재무, 예산, budget, finance, 회계, 매출, 비용, 손익 |
| 기타 | 위 키워드에 해당하지 않는 경우 |

저장 시 해당 카테고리 폴더가 프로젝트 내에 없으면 **자동 생성** (IndexedDB + 디스크 모두).

## 5. 파일 브라우저에서 문서 열기 (버그 수정)

### 근본 원인
파일 클릭 시 `selectFile()`만 호출하고 `content`를 에디터에 로드하지 않음.

### 변경 내용

| 파일 | 변경 내용 |
|------|-----------|
| `src/components/file-browser/FileBrowser.tsx` | 파일 선택 시 `file.content`가 있으면 `setEditorContent(file.content, file.name)` 호출 |

---

## 6. 프로젝트 내 폴더 생성 시 디스크 반영 (버그 수정)

### 근본 원인
`FileBrowser.handleCreateFolder()`와 `SaveToProjectDialog.handleCreateFolder()`가 IndexedDB에만 폴더 레코드를 생성하고, 실제 디스크에는 폴더를 만들지 않음.

### 변경 내용

| 파일 | 변경 내용 |
|------|-----------|
| `src/components/file-browser/FileBrowser.tsx` | 폴더 생성 시 `createSubFolderOnDisk(project.path, folderName)` 호출하여 디스크에도 폴더 생성 |
| `src/components/editor/SaveToProjectDialog.tsx` | 동일하게 디스크 폴더 생성 추가 |
