# DocuMind v0.4.5 Release Report

**날짜**: 2026-03-11
**버전**: 0.4.5
**주요 변경**: 바이너리 파일(docx/xlsx/pptx) 내용 기반 분류 추가

---

## 변경 사항

### 바이너리 파일 텍스트 추출 후 분류 (`classifyService.ts`)

#### 문제
- 이전: docx, xlsx, pptx, pdf 파일 업로드 시 **파일명만으로 분류**
- 파일명에 "보고서", "회의록" 같은 키워드가 없으면 분류 불가 → 무조건 "기타"

#### 해결: `extractTextFromBinary()` 함수 추가
- **docx**: mammoth 라이브러리로 텍스트 추출 (`extractRawText`)
- **xlsx**: SheetJS로 파싱 → 시트명 + 처음 50행의 셀 값을 평문으로 변환
- **pptx**: pptxService로 슬라이드 텍스트 추출

#### 분류 흐름 (개선 후)
```
파일 업로드
├─ 텍스트 파일 (txt, md, csv...) → 내용 직접 읽기
└─ 바이너리 파일
   ├─ docx → mammoth.extractRawText() → 처음 3000자
   ├─ xlsx → SheetJS 파싱 → 시트명 + 50행 텍스트
   ├─ pptx → 슬라이드 텍스트 추출 → 처음 3000자
   └─ pdf → (텍스트 추출 미지원, 파일명 + AI fallback)

→ smartClassify(파일명, 추출된텍스트)
  ├─ 규칙 키워드 매칭 → 카테고리 결정
  └─ 매칭 실패 → AI 분류 (설정된 경우)
```

#### 적용 위치
- `classifyAndAddWebFile()` — 웹 파일 선택 업로드
- `classifyAndAddTauriFile()` — Tauri 드래그앤드롭 업로드

#### 예시
| 파일 | 이전 | 이후 |
|------|------|------|
| `Q1실적.xlsx` (내용: 매출, 비용, 영업이익 컬럼) | 기타 | **재무** |
| `프로젝트A.docx` (내용: "회의 참석자: ..." ) | 기타 | **회의** |
| `신규서비스.pptx` (내용: "사업 전략", "로드맵") | 기타 | **기획** |
