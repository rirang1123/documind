# DocuMind v0.4.7 Release Report

**날짜**: 2026-03-11
**버전**: 0.4.7
**주요 변경**: NSIS 언인스톨러 데이터 삭제 + 재설치 데이터 연동 + 인앱 업데이트

---

## 변경 사항

### 1. NSIS 언인스톨러 — "애플리케이션 데이터 삭제하기" 강화

#### 문제
- 언인스톨 시 "애플리케이션 데이터 삭제하기" 체크해도 `Documents/DocuMind/` 폴더가 남아있음
- Tauri 기본 동작은 `%LOCALAPPDATA%/com.documind.app/`만 삭제

#### 해결: NSIS 훅 (`src-tauri/nsis-hooks.nsh`)
- `NSIS_HOOK_POSTUNINSTALL` 매크로 추가
- `$DeleteAppDataCheckboxState` 변수로 체크박스 상태 확인
- `$UpdateMode` 확인 — 자동 업데이트 시에는 데이터 삭제하지 않음
- 체크 시: `$DOCUMENTS\DocuMind` 폴더 재귀 삭제

#### 삭제되는 항목 (체크 시)
| 항목 | 경로 | 담당 |
|------|------|------|
| WebView2 데이터 (IndexedDB) | `%LOCALAPPDATA%\com.documind.app\` | Tauri 기본 |
| 앱 설정 | `%APPDATA%\com.documind.app\` | Tauri 기본 |
| 사용자 문서/프로젝트 | `Documents\DocuMind\` | **NSIS 훅 (신규)** |

---

### 2. 재설치 시 기존 데이터 자동 연동

#### 문제
- 재설치 후 IndexedDB에 저장된 프로젝트 경로가 디스크와 불일치할 수 있음
- 설치 경로 변경 시 프로젝트 폴더를 찾지 못함
- 기존 폴더가 있어도 새 폴더를 생성하지 못하는 충돌

#### 해결: `dataIntegrity.ts` 서비스 (`src/services/dataIntegrity.ts`)
- **`validateAndRepairProjectPaths()`** — 앱 시작 시 자동 실행
  1. 각 프로젝트의 `path`가 디스크에 존재하는지 확인
  2. 존재하지 않으면 현재 저장소 기본 경로(`Documents/DocuMind/`)에서 동일 이름 폴더 탐색
  3. 찾으면 경로 재매핑 (프로젝트 + 하위 파일 전체)
  4. 찾지 못하면 새 폴더 자동 생성
- **`discoverOrphanedFolders()`** — 디스크에 있지만 DB에 없는 고아 폴더 탐색 (향후 UI 연동용)
- `App.tsx`의 초기화 시퀀스에 `validateAndRepairProjectPaths()` 추가

#### 시나리오
| 상황 | 이전 동작 | v0.4.7 동작 |
|------|-----------|-------------|
| 재설치 후 동일 경로 | 작동 (우연) | 검증 후 정상 연결 |
| 재설치 후 경로 변경 | 프로젝트 접근 불가 | 자동 경로 재매핑 |
| IndexedDB만 남음 (폴더 삭제됨) | 오류 | 폴더 자동 재생성 |

### 3. 인앱 업데이트 시스템

#### 구성 요소
- **Rust**: `tauri-plugin-updater` 등록 (`lib.rs`)
- **서명 키**: Minisign Ed25519 키쌍 생성 (`src-tauri/.tauri-signing-key`)
- **설정**: `tauri.conf.json`의 `plugins.updater` (pubkey + endpoints)
- **서비스**: `src/services/updateService.ts` — 업데이트 확인/다운로드/설치
- **UI**: `src/components/settings/UpdateSection.tsx` — 설정 > 일반 탭

#### 업데이트 흐름
```
사용자: 설정 > "업데이트 확인" 클릭
  ↓
updateService.checkForUpdates()
  → check() (tauri-plugin-updater)
  → endpoints URL에서 latest.json 다운로드
  → 현재 버전과 비교
  ↓
업데이트 있음:
  → 버전, 릴리스 노트 표시
  → "업데이트 설치" 클릭
  → downloadAndInstall() + 진행률 표시
  → relaunch() (앱 재시작)
```

#### 설정 (tauri.conf.json)
- **pubkey**: Minisign 공개키 (서명 검증용)
- **endpoints**: GitHub Releases latest.json URL (변경 가능)
- **installMode**: `passive` (진행률 표시, 사용자 조작 불필요)

#### 보안
- 모든 업데이트는 Minisign 서명으로 검증
- TLS 강제 (HTTPS 엔드포인트만 허용)
- 서명 키 `.gitignore`에 추가 (커밋 방지)
