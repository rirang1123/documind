# DocuMind v0.4.8 Release Report

**날짜**: 2026-03-11
**버전**: 0.4.8
**주요 변경**: 문서 분류 엔진 대폭 개선 + 테스트 빌드 감지 + 앱 시작 시 자동 업데이트 체크

## 변경 사항

### 1. 문서 분류 엔진 대폭 개선 (`classifyService.ts`)

#### 문제
- 기존 분류는 단순 키워드 빈도 기반으로, 유사 문서 간 오분류가 잦았음
- "평가 보고서"와 "평가표", "회의 결과 보고서"와 "회의록" 등을 구분하지 못함

#### 해결: 5단계 분류 파이프라인 + 충돌 해소 로직

| 단계 | 방식 | 가중치 | 설명 |
|------|------|--------|------|
| 0단계 | 복합 구조 패턴 | ×8 | 2개 이상 구조 요소 동시 출현 (최고 신뢰도) |
| 1단계 | 단일 구조 패턴 | ×5~6 | 문서 골격의 확실한 단서 |
| 2단계 | 본문 키워드 빈도 | ×1 | 일반 키워드 매칭 |
| 3단계 | 제목 접미사 | ×3/6 | 파일명 끝의 명확한 유형명 |
| 4단계 | 제목 키워드 | ×0.3/2.0 | 최후의 보조 수단 |

#### 충돌 해소 (Disambiguation)
- **"주제(topic)"와 "형식(format)" 분리** — 형식이 분류 기준
- 예: "평가 보고서" → 주제=평가, 형식=보고서 → `report`
- 예: "회의 결과 보고" → 주제=회의, 형식=보고서 → `report`
- 6가지 카테고리 경합 쌍에 대한 형식 신호 판정기 구현

#### 분류 카테고리 (7종)
| 카테고리 | 설명 |
|----------|------|
| meeting | 회의록 |
| contract | 계약서 |
| report | 보고서 |
| planning | 기획서/계획서 |
| evaluation | 평가서 |
| finance | 재무 |
| reference | 참고자료/매뉴얼 |

#### 바이너리 파일 텍스트 추출 (`extractTextFromBinary`)
- docx → mammoth, xlsx → SheetJS, pptx → pptxService 활용
- 바이너리 파일도 내용 기반 분류 가능

### 2. 테스트 빌드 감지 시스템 (`buildEnv.ts`)

#### 신규 유틸리티
- **`isTestBuild()`** — 앱 이름에 'Test' 포함 여부로 테스트 빌드 판별
- **`getAppVersion()`** — Tauri API로 현재 버전 가져오기

#### 사이드바 표시 (`Sidebar.tsx`)
- 테스트 빌드 시 앱 이름 옆에 **주황색 "TEST" 배지** 표시
- 모든 빌드에서 **버전 번호** (예: `v0.4.8`) 표시

---

### 3. 앱 시작 시 자동 업데이트 체크 (`UpdateChecker.tsx`)

- v0.4.7에서는 설정 화면에서 수동으로만 업데이트 확인 가능
- **v0.4.8**: `UpdateChecker` 컴포넌트 구현 (아직 MainLayout에 미연결)
  - 앱 시작 시 자동으로 업데이트 확인
  - 화면 우하단 토스트 형태 알림
  - "지금 업데이트" / "나중에" 선택
  - 다운로드 진행률 바 표시
  - 설치 완료 후 "지금 재시작" 버튼

> **참고**: `UpdateChecker`는 구현 완료되었으나 MainLayout에 아직 import되지 않은 상태. 다음 버전에서 연결 예정.
