# DocuMind v0.1.8 릴리스 리포트

- **날짜**: 2026-03-10
- **버전**: 0.1.7 → 0.1.8
- **빌드**: DocuMind_0.1.8_x64-setup.exe / DocuMind_0.1.8_x64_en-US.msi

---

## 변경 사항

### 1. 한글 IME 입력 수정 (설정 > 커스텀 지시사항)

**이전**: textarea의 onChange에서 즉시 DB 저장 → 한글 조합 중 리렌더링으로 입력이 깨짐
**이후**: 로컬 state로 입력 관리, onBlur/onCompositionEnd 시점에만 DB 저장

| 파일 | 변경 내용 |
|------|-----------|
| `src/components/settings/SettingsView.tsx` | textarea에 `customText` 로컬 state 사용, `onBlur`/`onCompositionEnd`에서 저장 |

### 2. 문서 유형별 기본 틀(템플릿) 추가

**이전**: AI가 자유 형식으로 문서 생성 → 기본 구조 없이 내용이 빈약하거나 불완전
**이후**: 보고서/기획서/회의록/이메일/일반 각 유형별 표준 구조를 프롬프트에 포함, AI가 해당 틀에 맞춰 완성된 문서 생성

| 파일 | 변경 내용 |
|------|-----------|
| `src/services/ai-provider/documentAI.ts` | `DOCUMENT_TEMPLATES` 객체 추가 (5가지 유형별 구조), `generateDocument()` 프롬프트에 템플릿 구조 포함 |

### 3. 건너뛴 질문 반복 방지

**이전**: 사용자가 빈 답변(Enter)으로 건너뛴 질문과 비슷한 내용을 추가 질문에서 다시 질문
**이후**: 건너뛴 질문 키를 추적하여 `getFollowUpQuestions`에 전달, AI가 같은 주제를 다시 묻지 않음

| 파일 | 변경 내용 |
|------|-----------|
| `src/components/ai/AIAssistant.tsx` | `skippedKeys` state 추가, 건너뛴 질문 키 추적, follow-up 호출 시 전달 |
| `src/services/ai-provider/documentAI.ts` | `getFollowUpQuestions()`에 `skippedKeys` 매개변수 추가, 프롬프트에 건너뛴 주제 명시 |

### 4. 답변 없는 항목 문서 제외

**이전**: 정보 미제공 섹션에 "추후 보완 예정" 등의 빈 자리표시 포함
**이후**: 정보가 없는 섹션은 문서에서 완전히 생략하도록 프롬프트에 명시

| 파일 | 변경 내용 |
|------|-----------|
| `src/services/ai-provider/documentAI.ts` | `generateDocument()` 프롬프트에 "정보 없는 섹션 완전 생략" 지침 추가 |
