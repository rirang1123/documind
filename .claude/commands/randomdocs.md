test_docs 폴더에 30개의 .docx 파일을 무작위로 생성하세요.

## CTO (Chief Technical Orchestrator) 패턴

문서 생성을 **병렬 Agent 호출**로 수행하세요. 아래 단계를 순서대로 따르세요.

### Phase 1: 사전 준비 (직접 실행)

1. `pip install python-docx` 확인
2. 기존 test_docs 내 파일 삭제
3. test_docs 폴더 존재 확인

### Phase 2: 문서 설계 (직접 실행)

30개 문서의 **파일명 + 유형 + 도메인**을 먼저 설계하세요.

- 9가지 유형을 골고루 배분 (각 3~4개)
- 파일명은 주제만 반영 (유형 힌트 금지)
- 도메인은 IT, 마케팅, 인사, 물류, 금융, 제조, 교육, 법무, 의료, 에너지 등에서 다양하게

설계 결과를 6개 배치(Batch)로 나누세요 (각 5개 문서).

### Phase 3: 병렬 생성 (Agent 6개 동시 호출)

**반드시 6개 Agent를 하나의 메시지에서 동시에 호출**하세요.
각 Agent에게 5개 문서 생성을 맡기세요.

각 Agent 호출 시 아래 내용을 프롬프트에 포함하세요:

```
아래 5개의 .docx 문서를 python-docx로 생성하는 Python 스크립트를 작성하고 실행하세요.
작업 디렉토리: C:/Users/NSL/Desktop/project/project_3
저장 경로: test_docs/

[문서 목록]
1. 파일명: {파일명}.docx | 유형: {유형} | 도메인: {도메인}
2. ...

[문서 품질 요구사항]
- 각 문서는 2~3페이지 분량 (800~1500자 이상의 충분한 본문)
- 유형별 문서 구조를 사실적으로 반영:
  * 전략/기획: 배경→목적→추진전략→실행계획→기대효과→일정 구조
  * 보고서: 개요→현황분석→주요성과→이슈→향후계획 구조, 전년대비/달성률 등 수치 포함
  * 회의록: 일시/장소/참석자→안건→논의내용→결정사항→조치사항→다음회의 구조
  * 참고자료: 목적→적용범위→용어정의→세부지침→FAQ→부록 구조
  * 평가서: 평가개요→평가기준(배점표)→항목별평가→종합의견→개선권고 구조
  * 계약서: 제1조~제N조 조문 체계, 갑/을 당사자, 계약기간/금액/위약금/관할법원
  * 재무: 예산편성표, 집행현황, 합계/소계, 전년대비 증감, 비목별 상세
  * 기타: 위 유형에 해당하지 않는 자유 형식
- 표(Table)를 적극 활용하세요 (최소 1개 이상)
- 실제 기업 문서처럼 구체적인 수치, 날짜, 이름, 부서명 포함
- 날짜는 2024~2026년 사이 무작위
- 경계 사례를 포함: 재무 수치가 포함된 보고서, 평가 내용이 포함된 기획서 등
  (단, 문서의 핵심 형식/구조는 지정된 유형을 따라야 함)

[코드 규칙]
- 함수명은 doc_01, doc_02 등 번호만 사용
- 함수명, 변수명, 주석에 유형 힌트(보고서, 계약서 등)를 남기지 마세요
- python-docx 사용 (from docx import Document, from docx.shared import Pt, Cm, Inches, RGBColor)
- 스크립트 실행 후 스크립트 파일 삭제
- 파일 생성 완료 후 생성된 파일명 목록 출력
```

### Phase 4: 결과 확인 (직접 실행)

모든 Agent 완료 후:
1. test_docs 내 파일 목록과 총 개수 출력
2. 각 파일의 크기 확인 (너무 작은 파일 없는지)
3. 유형별 배분 현황 표시
